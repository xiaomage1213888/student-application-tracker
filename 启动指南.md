# 学生秋招投递记录管理系统 - 快速启动指南

## 项目结构

```
student-application-tracker/
├── backend/          # 后端服务
├── frontend/         # 前端应用
└── README.md
```

## 启动步骤

### 1. 启动后端服务

打开 PowerShell 或 CMD，执行：

```bash
cd backend
npm install
npm run dev
```

后端服务将在 http://localhost:3000 启动

### 2. 启动前端服务

打开新的 PowerShell 或 CMD 窗口，执行：

```bash
cd frontend
npm install
npm run dev
```

前端应用将在 http://localhost:5173 启动

### 3. 访问系统

浏览器打开：http://localhost:5173

**默认老师账号**：
- 用户名：admin
- 密码：admin123

## 功能说明

### 学生端
- 注册/登录：需要填写姓名和班级，系统会验证是否在授权名单中
- 投递记录管理：添加、编辑、删除投递记录
- 个人看板：查看投递统计、进展分布、趋势图表

### 老师端
- 学生名单管理：导入 Excel/CSV 授权名单
- 班级数据查看：按班级查看学生投递情况
- 数据统计看板：整体统计、班级排名、进展分布
- 投递记录导出：支持导出 Excel 文件

## Excel 导入格式

创建 Excel 文件，包含以下列：
- name: 学生姓名
- className: 班级名称

示例：
```
name,className
张三，2024 级计算机 1 班
李四，2024 级计算机 1 班
王五，2024 级软件工程 2 班
```

## 技术栈

- **前端**：Vue 3 + Element Plus + TypeScript + Vite + ECharts
- **后端**：Node.js + Express + TypeScript + Sequelize
- **数据库**：SQLite（开发环境）
- **认证**：JWT Token

## 常见问题

### 1. npm install 失败
- 检查 Node.js 版本（建议 16+）
- 尝试使用淘宝镜像：`npm config set registry https://registry.npmmirror.com`

### 2. 后端启动失败
- 检查 3000 端口是否被占用
- 确保 backend/.env 文件存在

### 3. 前端无法连接后端
- 确保后端服务已启动
- 检查前端代理配置（vite.config.ts）

## 数据库切换（生产环境）

如需使用 MySQL，修改 `backend/.env`：

```env
DB_DIALECT=mysql
DB_HOST=localhost
DB_PORT=3306
DB_NAME=student_tracker
DB_USER=root
DB_PASSWORD=yourpassword
```
